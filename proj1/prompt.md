# 네이버 기자 분석 웹앱 개발 프롬프트

## 개요
포스코 홍보 담당자가 네이버 뉴스에서 매체명과 기자를 검색하고 기사를 분석하여 미팅 토킹포인트를 자동 생성하는 Flask 웹 애플리케이션 개발.

---

## 기술 스택
- **백엔드**: Python + Flask
- **크롤링**: Playwright (Chromium headless) + BeautifulSoup4
- **AI 분석**: OpenAI API (`.env`의 `OPENAI_API_KEY`, 모델: `gpt-5.5`)
- **프론트엔드**: Bootstrap 5 (PC + 모바일 반응형)
- **포트**: 5001

---

## .env 파일 (기존 파일 사용)
```
FLASK_APP=app.py
FLASK_ENV=development
SECRET_KEY=change-this-secret
OPENAI_API_KEY=<키값>
OPENAI_MODEL=gpt-5.5
REQUEST_TIMEOUT=12
MAX_ARTICLES=30
```

### 3. AI 분석 — OPEN AI 2회 분리 호출

**1차 호출**: 최신 기사 30건 → 아래 항목 분석
- 기자 최근 관심 분야 (3~5개)
- 관련 기사 및 논조 (전체 요약, 특징, stance: 비판적/중립적/우호적)
- 최근 관심 키워드 (5~10개, 빈도: 높음/중간/낮음)

**2차 호출**: 최신 기사 30건 + 포스코 관련 기사 5건 → 아래 항목 생성
- 토킹포인트 정확히 **5개** (제목, 내가 기자에게 직접 말하는 1인칭 대화 시나리오 200자 이내, 활용 이유)
- 전반적인 미팅 전략
- 기사 본문 미수집 시 제목만으로도 분석
