# 네이버 기자 분석 웹앱 개발 프롬프트

## 1. 개발 목표

포스코 홍보 담당자가 **네이버 뉴스 기자홈**에서 특정 매체의 기자를 검색하고, 해당 기자의 최신 기사와 포스코 관련 기사를 분석하여 **미팅용 토킹포인트 5개**를 자동 생성하는 Flask 웹 애플리케이션을 개발한다.

---

## 2. 기술 스택

- 백엔드: Python + Flask
- 크롤링: Playwright Chromium Headless + BeautifulSoup4
- AI 분석: Google Gemini API
- 모델: `.env`의 `GEMINI_MODEL`, 기본값 `gemini-2.0-flash`
- 프론트엔드: Bootstrap 5
- 반응형: PC + 모바일 지원
- 실행 포트: `5001`

---

## 3. `.env` 설정

기존 `.env` 파일을 사용한다.

```env
FLASK_APP=app.py
FLASK_ENV=development
SECRET_KEY=change-this-secret
GOOGLE_API_KEY=<키값>
GEMINI_MODEL=gemini-2.0-flash
REQUEST_TIMEOUT=12
MAX_ARTICLES=30
```

## 5. 사용자 입력

메인 화면에서 다음 2개 항목을 입력받는다.

- 매체명
- 기자명

입력된 매체명을 기준으로 사전에 정의된 `officeId`에 매핑하고, 해당 네이버 기자홈 URL로 바로 이동하여 기자를 탐색한다.

## 7. 기자 검색 로직

해당 매체의 기자 목록 페이지는 무한 스크롤 방식이므로 다음 방식으로 기자를 찾는다.

1. `https://media.naver.com/journalists/whole?officeId={officeId}` 접속
2. 최대 10회 스크롤
3. 기자명 탐색
4. 매칭 우선순위 적용

매칭 우선순위는 다음과 같다.

1. 기자 URL이 존재하고 이름이 정확히 일치
2. 기자명이 포함 일치
3. 페이지 텍스트에 기자명이 포함

기자를 찾지 못하면 오류 메시지를 반환한다.

## 8. 기사 수집 로직

기자 페이지에 접속한 뒤 다음 과정을 수행한다.

1. `전체 기사` 탭 클릭
2. 최대 5회 스크롤
3. 기사 목록 수집
4. 최신 기사 최대 20건 수집
5. 포스코 관련 기사 최대 5건 수집

포스코 관련 기사 판별 키워드는 다음과 같다.

```python
POSCO_KEYWORDS = [
    "포스코",
    "POSCO",
    "포항제철",
    "포스코홀딩스",
    "포스코퓨처엠",
    "포스코인터내셔널",
    "포스코건설",
    "포스코케미칼"
]
```

기사 원문 수집 선택자 우선순위는 다음과 같다.

```text
div#dic_area
→ div.newsct_article
→ div#articleBodyContents
→ article
```

본문 수집이 실패한 경우에도 제목, 날짜, URL만으로 분석을 진행한다.

## 9. AI 분석 방식

Gemini API를 2회 분리 호출한다.

### 1차 분석

입력 데이터:

- 최신 기사 20건

분석 항목:

- 관심 분야 3~5개
- 기사 논조
  - 전체 요약
  - 특징
  - stance: 비판적 / 중립적 / 우호적
- 최근 관심 키워드 5~10개
  - 키워드명
  - 빈도: 높음 / 중간 / 낮음

---

### 2차 분석

입력 데이터:

- 최신 기사 20건
- 포스코 관련 기사 5건

생성 항목:

- 포스코 보도 성향
  - 논조
  - 주요 토픽
  - 요약
- 토킹포인트 정확히 5개
  - 제목
  - 기자에게 직접 말하는 1인칭 대화 시나리오
  - 100자 이내
  - 활용 이유
- 전반적인 미팅 전략

## 10. Flask 앱 구조

크롤링과 분석은 시간이 오래 걸리므로 백그라운드 스레드로 실행한다.

```python
threading.Thread
```

진행 상황은 프론트엔드에서 2초 간격으로 폴링한다.

```text
/api/status/<job_id>
```

필수 라우트는 다음과 같다.

```text
GET  /                    메인 검색 화면
POST /search              검색 작업 생성
GET  /progress/<job_id>   진행 상황 화면
GET  /result/<job_id>     분석 결과 화면
GET  /api/status/<job_id> 진행 상태 API
```

---

## 11. 결과 화면 구성

결과 화면은 Bootstrap 탭 4개로 구성한다.

| 탭 | 내용 |
|---|---|
| 종합 분석 | 관심 분야, 기사 논조, 최근 관심 키워드 |
| 최신 기사 | 최신 기사 20건의 제목, 날짜, 원문 링크 |
| 포스코 기사 | 포스코 관련 기사 5건, 포스코 보도 성향 요약 |
| 토킹포인트 | 미팅 전략, 토킹포인트 5개 |

## 12. 핵심 개발 조건

- 포트는 반드시 `5001` 사용
- 매체 검색은 네이버 검색이 아니라 `officeId` 기반 직접 접근 방식 사용
- 무한 스크롤 대응
- 기자 검색 실패, 매체명 오류, 기사 본문 수집 실패에 대한 예외 처리 포함
- 본문이 없어도 제목 기반 분석 가능해야 함
- Gemini API 호출은 2회로 분리
- 토킹포인트는 반드시 5개 생성
- 모바일에서도 사용 가능한 반응형 UI 구현
- `.env`의 `GOOGLE_API_KEY`, `GEMINI_MODEL`을 사용
