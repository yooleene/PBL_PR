현재 `pr` 폴더 하위에는 Flask로 개발된 여러 개의 앱이 존재함.

구조 예시:
- pr/
    - proj1/
        - app.py
    - proj2/
        - app.py
    - proj3/
        - app.py
    - index.html

현재는 각 폴더의 `app.py`가 개별적으로 실행되며,
도메인 또는 URL을 통해 각각 독립적으로 접근 가능한 상태임.

========================
목표
========================

보안성과 운영 효율성을 위해 현재의 다중 Flask 앱 구조를
하나의 Flask 애플리케이션으로 통합하고 싶음.

기존 앱의 독립성을 최대한 유지하면서도,
운영 시에는 하나의 Flask 프로세스로 실행되도록 구조를 개선해줘.

가능하면 Flask Blueprint 구조를 사용하여
proj1, proj2, proj3를 각각 모듈화해줘.

========================
핵심 요구사항
========================

1. Flask 앱 통합
- proj1, proj2, proj3 앱을 하나의 Flask 애플리케이션으로 통합
- 운영 시 하나의 프로세스로 실행
- Flask Blueprint 기반 구조 권장
- 각 프로젝트는 `/proj1`, `/proj2`, `/proj3` 경로로 접근 가능해야 함
- windowsOS, macOS 로컬 개발과 Linux VPS 호스팅을 모두 지원해야 함

2. 독립 실행과 통합 실행 모두 고려
- 개발 중에는 각 프로젝트를 단독 실행할 수 있는 구조도 고려
- 운영 시에는 통합 Flask 앱으로 실행
- 예:
  - 통합 실행: `python app.py`
  - 개별 테스트: proj1만 별도 실행 가능하도록 구조 제안

3. 메인 페이지 구성
- 로그인 성공 후 접근 가능한 메인 페이지로 사용
- 메인 페이지에는 proj1, proj2, proj3로 이동하는 버튼 또는 메뉴가 있어야 함
- 메인 페이지에서 버튼 클릭 시 현재 페이지 이동 방식이 아니라 새 브라우저 창(또는 새 탭)으로 열리도록 구성
- 예:
  - proj1 버튼 클릭 → 새 창에서 `/proj1`
  - proj2 버튼 클릭 → 새 창에서 `/proj2`
  - proj3 버튼 클릭 → 새 창에서 `/proj3`
- `target="_blank"` 또는 `window.open()` 방식 사용 가능
- 메인 페이지는 유지된 상태에서 각 프로젝트가 새 창으로 열리도록 구현
- 새 창에서도 Flask session이 유지되어 로그인 상태가 공유되어야 함
- 로그인되지 않은 상태에서 새 창 접근 시 `/login` 페이지로 이동해야 함

4. 로그인 필수 조건
- 메인 페이지도 반드시 로그인 후 접근 가능해야 함
- 모든 프로젝트 페이지도 반드시 로그인 후 접근 가능해야 함
- 사용자가 `/`, `/proj1`, `/proj2`, `/proj3` URL을 직접 입력해도 로그인하지 않은 상태라면 `/login` 페이지로 이동해야 함
- 로그인하지 않은 사용자는 어떤 내부 페이지도 볼 수 없어야 함

5. 로그인 기능
- 관리자(admin)와 일반 사용자(user) 계정을 구분
- 로그인 성공 시 session에 사용자 정보 저장
- session 기반으로 로그인 상태 유지
- 로그아웃 기능 포함

6. 권한 관리
- 관리자와 일반 사용자 권한을 명확히 구분
- 향후 특정 메뉴 또는 기능별 접근 제한이 가능하도록 설계
- `login_required`, `admin_required` 같은 데코레이터 구조 제안

7. proj2 권한 조건
- proj2에는 데이터 추출 기능과 추출 후 저장 기능이 있음
- 해당 기능은 반드시 admin 계정만 사용할 수 있어야 함
- user 계정은 proj2에 접근은 가능하지만,
  저장되어 있는 내용을 조회만 가능해야 함
- user 계정에서는 데이터 추출 버튼, 저장 버튼, 실행 기능이 비활성화되거나 접근 차단되어야 함
- URL을 직접 입력해서 데이터 추출/저장 기능에 접근하려 해도 admin이 아니면 차단되어야 함

8. 환경 변수 관리
- 모든 계정 정보 및 환경 설정은 `pr/.env` 파일에서 관리
- 비밀번호, SECRET_KEY 등을 코드에 직접 하드코딩하지 않음

.env 예시 포함:
- ADMIN_ID
- ADMIN_PASSWORD
- USER_ID
- USER_PASSWORD
- SECRET_KEY
- FLASK_ENV
- FLASK_DEBUG

9. 보안 강화
- Flask session 기반 인증 사용
- `.env` 기반 설정 로딩
- `.gitignore`에 `.env` 포함
- 비밀번호를 코드에 직접 작성하지 않음
- 모든 내부 라우트에 로그인 검증 적용
- admin 전용 기능은 서버 라우트에서도 반드시 권한 검증
- 단순히 버튼을 숨기는 방식만으로 보안 처리하지 말 것
