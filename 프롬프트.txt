ChatGPT와 대화하며 기초 프롬프트 작성
: 특정 기자의 기사를 추출하기 위한 로직 : 네이버 검색으로 기사 검색 - 해당 기자가 맞는지 확인 - 기자 링크를 활용하여 해당 기자홈으로 이동
* 네이버 API에는 기자명 없음
* 기자홈에는 검색 기능이 없어 오히려 검색 경로가 복잡해짐
------------------------------------

너는 시니어 Python 엔지니어이자 웹 데이터 수집 아키텍트다.

내가 만들고 싶은 것은 “네이버 기자 기사 분석 앱”의 프로토타입이다.

[목표]
사용자가 앱에서 아래 값을 입력하면:
- 언론사명
- 기자명
- 조회 시작일
- 조회 종료일
- 최신 기사 20건 수집

다음 순서로 동작해야 한다:
1. 네이버에서 "언론사명 + 기자명 + 기자" 조합으로 기사 후보를 검색한다.
2. 검색 결과 기사 본문으로 들어가서, 해당 기자가 맞는지 검증한다.
3. 검증된 기사에서 기자명 링크를 추출해 기자 페이지 URL을 확보한다.
4. 기자 페이지에서 해당 기자의 기사 목록을 수집한다.

중요:
- 나는 이것을 “웹 앱”으로 사용할 것이므로, 코드가 실제 서비스 프로토타입 구조에 가깝게 작성되어야 한다.
- Python 기반으로 작성한다.
- Flask를 사용한다.
- 데이터 수집은 requests + BeautifulSoup를 우선 사용하되, 동적 페이지 대응이 필요하면 Playwright를 fallback 옵션으로 설계한다.
- DB는 우선 SQLite 또는 PostgreSQL 둘 다 쉽게 바꿀 수 있도록 SQLAlchemy 구조로 만든다.
- 코드 구조는 모듈화한다.
- 예외처리, 재시도, 타임아웃, 로깅을 포함한다.
- robots.txt / 이용약관 / 과도한 요청 방지에 대한 주석과 주의사항을 코드 및 README 수준으로 반드시 명시한다.
- 실제 서비스 운영 시 법적/정책적 검토가 필요하다는 경고를 포함한다.
- 무한 스크롤, 페이지네이션, HTML 구조 변경 가능성을 고려해 selector를 상수화하고 유지보수 가능하게 작성한다.




1.핵심 수집 로직 구현
반드시 아래 로직을 코드로 반영하라.

[A] 기자 후보 기사 검색
- 검색어는 기본적으로 "{언론사명} {기자명} 기자" 사용
- 보조 검색어:
  - "{기자명} 기자 {언론사명}"
  - "{언론사명} {기자명}"
- 검색 결과 중 뉴스 기사 URL만 우선 수집
- 중복 URL 제거

[B] 기사 본문 검증
각 기사에 대해 아래를 파싱:
- 제목
- 본문
- 발행일
- 언론사명
- 기자명
- 기자 페이지 링크
검증 규칙:
- 기자명 정확히 일치 또는 공백/표기 변형 허용 범위 내 일치
- 언론사명 일치
- 기자 링크가 존재하면 신뢰도 가산
- 조건을 만족한 기사만 verified 처리

[C] 기자 페이지 확보
- 검증된 기사에서 기자 링크를 우선 추출
- 여러 기사에서 동일 기자 페이지 URL이 반복되면 그 URL을 대표값으로 채택
- 기자 페이지 URL이 확보되지 않으면 fallback 전략을 적용:
  1) 검증 기사 수가 충분하면 그 기사 집합만 분석
  2) 가능하면 officeId 등 추가 단서를 파싱해 기자 디렉토리 진입 시도
- 이 fallback 전략도 코드에 포함

[D] 기자 페이지 기사 목록 수집
- 기자 페이지에서 기사 제목, URL, 날짜, 카테고리 등 수집
- 페이지네이션 처리
- 조회 기간(date_from~date_to) 필터링
- limit 적용
- 중복 제거

[E] 최종 기사 재검증
- 기자 페이지에서 가져온 기사도 다시 기사 본문으로 접속해 기자명/언론사/날짜 검증
- 오탐 제거

- 초기 버전에서는 간단한 토큰화/불용어 제거/빈도 분석 기반으로 작성
- 코드에 “향후 KoNLPy, Kiwi, SentenceTransformer, LLM 요약으로 확장 가능”이라고 주석으로 남길 것

6. 매우 중요
- 절대 설명만 하지 말고, 실제 파일별 코드 내용을 작성하라.
- 가능한 한 바로 실행 가능한 형태로 작성하라.
- 응답 형식은 “프로젝트 파일별 코드” 중심으로 구성하라.
- 각 파일은 파일명 헤더를 달고 그 아래 전체 코드를 제공하라.
- 생략하지 말고 끝까지 출력하라.
- placeholder를 최소화하라.
- 테스트용 샘플 입력도 포함하라.

[구현 방향에 대한 추가 힌트]
- 네이버 기사 HTML 파싱과 기자 페이지 링크 추출이 핵심이다.
- “검색 → 기사 검증 → 기자 페이지 확보 → 기자 페이지 기사 수집 → 재검증 → 분석” 구조를 엄격히 유지하라.
- 첫 실행은 검색 기반, 이후 동일 기자는 DB에 저장된 reporter_page_url을 재사용하는 전략을 반영하라.
- 동일 기자 재분석 시 증분 수집이 가능하도록 구조를 열어두어라.

이제 위 요구사항을 만족하는 전체 프로젝트 코드를 작성하라.

기능 확장 프롬프트
-----------------
1.기자 페이지 url은 클릭하면 연결되도록 해줘
2.해당 기자의 최신 기사를 20건 보여줘
3.최신 기사 20건의 본문 내용을 앞에서 500자를 추출해서 각 기사 제목 아래에 출력해줘
  * 최신 기사 중에서 '단독' or '기획' or '르포' or '기자수첩' or '논설' or '사설' or '오피니언' 등은 별도 표시
4.해당 기자가 작성한 포스코그룹 관련 기사(최신순 10건)를 보여줘
5.해당 기자가 작성한 기사 중 댓글 반응이 좋은 기사(최신순, 5건)를 보여줘
6.다음 사항을 .env에 있는 GOOGLE_API_KEY를 사용하여 분석하고 화면에 출력해줘
 - 해당 기자의 관심분야, 기사논조
 - 기자와 미팅시 토킹포인트(포스코그룹 사업회사별 기자 관심사를 반영한 대화 주제 및 이슈에 대해 내용으로 생성)
