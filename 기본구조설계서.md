# AI 애플리케이션 기본 구조 설계서  

---

# 1. 프로젝트 개요

- 과제명: 그룹 홍보 업무 효율화를 위한 미디어 인텔리전스 AI Agent 개발
- 목적: 기자 정보·기사 성향·관심 이슈를 분석하여 홍보 담당자의 대응 역량 강화
- 개발 형태: Flask 기반 Web App
- 개발 환경: Python + Flask + HTML Template
- AI/LLM: Google Gemini API
- 데이터 수집: 네이버 뉴스 API
- 환경 설정: `.env` 기반 관리
- 유지보수 방향: 초보자도 수정 가능한 단순 구조 중심 설계

---

# 2. 서비스 구조 개요

```text
사용자(UI)
   ↓
Flask Web Application
   ↓
서비스 로직(Service Layer)
   ├─ 기자 정보 수집
   ├─ 기사 데이터 분석
   ├─ 키워드 분석
   ├─ 논조 분석
   └─ 토킹포인트 생성
   ↓
LLM(Gemini API)
   ↓
결과 출력(Web UI)
```

---

# 3. 애플리케이션 구조 설계

## 3-1. 화면 구성

| 화면 | 주요 기능 |
|---|---|
| 메인 화면 | 기자명/매체명 입력 |
| 분석 결과 화면 | 기자 프로필 및 기사 분석 결과 출력 |
| 토킹포인트 화면 | 포스코그룹 사업회사 관점 대화 주제 제공 |
| 관리자 설정 화면 | API Key 및 환경 설정 |

---

## 3-2. 사용자 입력 항목

| 입력 항목 | 설명 |
|---|---|
| 기자명 | 분석 대상 기자 |
| 매체명 | 소속 언론사 |

---

## 3-3. 출력 항목

| 출력 항목 | 설명 |
|---|---|
| 기자 프로필 | 직책, 출입분야, 경력 등 |
| 최신 기사 | 최근 기사 30건 |
| 관심 분야 | 기자 주요 관심 주제 |
| 기사 논조 | 긍정/부정/중립 분석 |
| 핵심 키워드 | 최근 기사 기반 키워드 |
| 관련 기사 | 포스코그룹 관련 기사 |
| 토킹포인트 | 사업회사 관점 대화 전략 |

---

# 4. 기능 구조 설계

## 4-1. 데이터 수집 기능

### 역할
- 네이버 뉴스 API 호출
- 기자 기사 데이터 수집
- 기사 제목/본문 저장

### 처리 흐름

```text
기자명 입력
→ 네이버 API 검색
→ 기사 데이터 수집
→ JSON 형태 저장
```

---

## 4-2. 기사 분석 기능

### 역할
- 기사 키워드 분석
- 기자 관심사 추출
- 기사 논조 분석

### 분석 항목

| 분석 항목 | 설명 |
|---|---|
| 관심 분야 | 반복 등장 주제 |
| 핵심 키워드 | 최근 기사 기반 추출 |
| 논조 분석 | 긍정/부정/중립 |
| 기사 유형 | 단독/사설/기획 등 |

---

## 4-3. 토킹포인트 생성 기능

### 역할
- Gemini API 기반 생성형 응답
- 사업회사별 대응 메시지 생성

### 처리 흐름

```text
기사 분석 결과
→ Gemini Prompt 생성
→ 사업회사 관점 분석
→ 토킹포인트 생성
```
