#  정부 안전/노동 관련 주요인사 발언 및 중대재해 분석 웹앱 개발 프롬프트

## 개요
정부의 주요인사가 발언하는 안전, 노동과 관련된 뉴스를 수집하고 이를 분석하는 Flask 웹 애플리케이션 개발.

---

## 기술 스택
- **백엔드**: Python + Flask
- **데이터 수집**: OpenAI Responses API `web_search` 도구 (`.env`의 `OPENAI_API_KEY`, 검색 모델: `gpt-4.1-mini`)
- **AI 분석**: OpenAI API (`.env`의 `OPENAI_API_KEY`, 모델: `gpt-4o-mini`)
- **프론트엔드**: Bootstrap 5 (PC + 모바일 반응형)
- **포트**: 5001

---

## .env 파일 (기존 파일 사용)
```
FLASK_APP=app.py
FLASK_ENV=development
SECRET_KEY=change-this-secret
OPENAI_API_KEY=<키값>
OPENAI_MODEL=gpt-4o-mini
OPENAI_WEB_SEARCH_MODEL=gpt-4.1-mini
OPENAI_WEB_SEARCH_CONTEXT_SIZE=medium
REQUEST_TIMEOUT=12
```

## 기능 요구사항

### 1. 데이터 수집
- OpenAI Responses API의 `web_search` 도구를 사용하여 최신 웹 검색을 수행
- 프롬프트는 2개로 별도로 사용하여 전달
- 프롬프트1: "이재명 대통령과, 김영훈 노동부장관이 안전 노동과 관련해 해당 기간에 발언한 내용을 언론이나 소셜미디어(유튜브, X, 페이스북 등)에 나온 내용을 모두 알려주고 출처도 명확히 알려줘"
- 프롬프트2: "해당 기간에 일어난 중대재해 사고 관련해서도 모두 찾아줘."
- 프롬프트 내의 "해당 기간"은 사용자가 별도로 입력해서 프롬프트에 들어가게 해주는데 버튼을 누르면 달력이 나와서 클릭하면 선택되도록 만들어줘
- 프롬프트1과 프롬프트2는 별도로 사용하여 데이터를 추출
- 프롬프트1인 주요인사 데이터는 1개의 기사가 하나의 데이터임
- 프롬프트2인 중대재해 데이터는 1개의 기사를 찾았으면 관련 재해 기사도 검색하여 데이터로 추출해줘
- 수집된 데이터는 누적 관리 될 수 있도록 데이터베이스에 저장해줘. 다음에 수집한 데이터는 누적으로 저장하고 담당자가 불필요한 내용은 삭제하거나 편집할거야.

### 2. 화면 출력
1. 화면구성(탭으로 구분) : 종합분석, 주요인사발언, 중대재해사례, 당사사고 대응방향
 1)종합분석 : OPENAI_API_KEY 사용하여 주요인사발언 화면과 중대재해화면의 내용을 종합적으로 분석한 요약 화면으로 출력하되 수직형태로 위부터 차례대로 아래로 내여가면서 출력
  - 주요 시사점 : 추출한 주요인사발언, 중대재해사례 데이터를 합쳐서 OPENAI_API에 전달하여 과거의 내용등을 종합적으로 고려했을때 현재 최근 노동, 안전과 관련한 정책적 변화가 어떤지 이에 대해서 포스코가 대응해야하는 방향성이 무엇인지를 요청하여 다음 샘플과 유사한 형태로경영층에 보고할 내용을 문장으로 작성
  --------------------------------------------------------------------------
    샘플:
      정부의 정책 초점이 '대기업 규제'에서 '중소규모 사업장 지원 및 예방'으로 이동
        - 정부는 큰 사업장은 CEO의 관심도도 높아지고 안전 관리 체계가 이제 어느 정도 
          안착되었다고 판단하는 듯하며, 50인 미만 소규모 사업장들의 중대재해 지원 및 예방에 초점
        - 산재 사고의 사각지대인 영세 사업장에 대해 '질타'보다는 '재정 지원(최대 90%) 및 점검' 
          중심의 정책을 펼치고 있으며. 인공지능(AI) 활용 중대재해 감축에도 관심.
      ☞ 당사 AI 안전기술을 활용해 정부의 관심사인 소규모 사업장의 안전을 강화하는 상생활동 시 효과 高
  ---------------------------------------------------------------------------
   * 주요 분석 포인트 : 대통령, 노동부장관 등이 언론, 뉴미디어를 통해 발언한 내용을 바탕으로 정부의 최근 안전/노동과 관련한 정책적 변화가 어떻게 이루어지고 있는지 분석을 하고. 타사 주요 중대재해/산업재해 사례는 어떤게 있었는지 이에 대해서 대통령/노동부장관이 언급한 내용이 있었는지를 기본적으로 분석
  - 포스코그룹 대응방안 : 주요 시사점에서 분석한 내용을 바탕으로하여 포스코그룹은 안전/노동과 관련해서 어떤 방향성으로 대응을 해야하는지 대응방안을 도출하여 출력
  - 주요인사발언 탭의 데이터 중 최근 주요인사 발언 5건 요약 출력
  - 중대재해사례 탭의 데이터 중 최근 중대재해 5건 요약 출력

 2)주요인사발언(데이터 수집의 프롬프트1 수집 데이터): 표 형태로 날짜/대상인물.기관/발언장소/주요발언내용/핵심키워드/주요발언기사출처
 3)중대재해사례(데이터 수집의 프롬프트2 수집 데이터): 표 형태로 순번/회사명/사고일/사고내용/대외대응/시사점/사과문
  * 사과문 : 담당자가 직접 입력할 수 있는 비어있는 열 생성(입력/수정 버튼 필요)
 4)당사사고 대응방향 : OPENAI_API_KEY에 전달하여 생성
  - 포스코에서 재해가 발생할 경우 담당자가 앱에서 재해내용을 입력할 수 있도록 해줘
  - 재해내용 입력 후 "대응방향" 버튼을 누르면, 입력된 당사의 중대재해에 옆에 당사의 중대재해에 대한 사과문을 생성해줘.
  - 사과문 생성시 참고내용 : 누적 관리된 주요인사발언, 중대재해사례 데이터를 참고하여 사과문을 작성하되, 기존에 생성되어 저장된 포스코 재해에 대한 사과문과 3)에서 담당자가 입력한 사과문운 참조해서 정부의 최근 주요 워딩을 차용하여 언론에 배포할 사과문을 생성해줘
  - 재해내용 입력한 하단에, 최근 정부의 안전 정책 동향을 고려한 당사 중대재해와 관련하여 회사의 대응 방향도 생성해줘

### 기능구현 추가 요구사항
- 주요인사발언, 중대재해사례 데이터는 앱에서 추가/수정/삭제 가능하고,csv 파일로 다운 가능하게 구현
- 맥북과 리눅스 가상서버 호스팅에서 동시에 사용 가능하도록 구현

## 아래부터는 초기 구현 후 추가로 수정했던 프롬프트임
-------------------------------------------

1차
-------------------------------------------

종합분석에서는 csv 다운은 필요 없음
데이터 추출하고 데이터베이스에 저장은 담당자가 버튼을 누르면 저장하도록 해줘.
종합분석의 주요 시사점과 포스코그룹 대응방안에 'title': , 'content': ,'direction':  등은 안나오게 해주고, 텍스트만 예쁘게 출력해줘
종합분석의 최근 주요인사 발언 5건과 최근 중대재해 5건의 제목에 링크를 연결하여 누르면 기사로 넘어갈 수 있도록 해줘
주요인사발언 탭과 중대재해사례 탭에서 사용자가 데이터를 추가하는 기능은 삭제해줘. CSV 다운로드는 유지.
중대재해사례 탭에 주요인사발언 탭의 주요발언기사출처 처럼 출처 링크를 추가해줘.
당사사고 대응방향 탭에서 대응방향 버튼을 눌러서 생성된 내용을 바로 저장하지 말고 사용자가 저장 버튼을 누르면 저장하게 해주고 수정도 가능하게 해줘



2차
-------------------------------------------
주요인사발언 탭과 중대재해사례 탭에 대한 데이터 추출은 별도로 진행해주고, 데이터 추출 버튼도 각 탭에 나오게 해줘.
담당자가 버튼을 누르면 저장하는 기능도 각 탭에 별도로 나오게 해줘.

3차
-------------------------------------------
종합분석 탭의 최근 주요인사 발언 5건과 최근 중대재해 5건의 링크가 연결이 안되는게 많아. URL이 정확확게 나오도록 수정해줘.

4차
-------------------------------------------
중대재해사례 탭에서 기사출처를 시사점 다음으로 순서를 변경해줘.



5차
-------------------------------------------
새로운 탭을 아래 내용처럼 추가해줘
- 탭 제목 : 데이터검색
- 기능 : 자연어로 입력을 하면 현재 앱에 저장된 데이터베이스 내에서 데이터를 검색하고 입력한 자연어에 적절하게 대답을 해줘

6차
-------------------------------------------
주요인사발언 탭과 중대재해사례 탭에 CSV 업로드 기능을 추가하고, 업로드한 CSV 내용을 앱 데이터베이스에 누적 저장해줘.

7차
-------------------------------------------
주요인사발언에 유튜브의 경우에는 주요발언기사 출처 링크가 삽입이 안되어 있어. 신문기사 외에도 출처를 신문기사와 같이 삽입해서 넣어줘.
